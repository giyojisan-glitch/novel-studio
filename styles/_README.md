# 风格包（Style Packs）

每个 `.md` 文件是一个风格包，文件名必须和 genre 对应（`科幻.md` / `仙侠.md` 等）。

## 工作方式

L3 段落写作的 system prompt = 通用风格基础（写在 `prompts.py` 的 `L3_SYSTEM_BASE`）+ 本目录的风格包文件正文。

修改本目录里的 `.md` 文件 → **立即生效**（下次 `novel-studio step` 就用新风格），无需改代码。

## 如何写风格包

模板建议：
1. **第一行**用 `【XX 风格】` 标注，方便 LLM 定位
2. **必要元素**：列出这个类型不可少的具象词汇
3. **爽点/冲突公式**：节奏规律（如"被轻视 → 反转 → 打脸"）
4. **人物语言风格**：对话的调性
5. **开场/结尾节奏**：章节结构
6. **禁忌**：不要写成什么样

越具体越好。"冷峻"不如"用传感器读数和时间戳构建张力"。

## 自定义风格包（V2 计划）

未来会支持：
- `--style 自定义风格名` CLI 参数（覆盖 genre 默认）
- 风格包组合：`--style 仙侠,黑色幽默`
- 灵感库 RAG 风格（把喜欢的小说塞进风格包目录下的子文件夹）
